# Custom SQL

:::tip Goal
Cases may arise that `ano` rule logic cannot address. Use custom sql-statements alongside tasks and rules to handle these cases.

- `where` allows you to explicitly write the content of a WHERE clause for a table select statement.
- `sql-before` allows you to write full CRUD statements that are run before tasks are run.
- `sql-after` allows you to write full CRUD statements that are run after tasks are run.

:::

## Use cases for custom sql-statements

All `ano` logic translates directly to sql in code.
