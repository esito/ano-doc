# `file`

:::tip Goal
Write `mask` rule logic to

- generate strings from file

:::

## Code Structure

<Tabs
  defaultValue="one"
  values={[
    { label: 'one file input', value: 'one' },
	{ label: 'multiple file inputs', value: 'multiple' },
  ]}
>

<TabItem value="one">

```ano
        ...
        mask <column> <rule-name>
            format %s
            file src/main/resources/<file-name.txt> random-order // random-order is optional
        ...
```

</TabItem>

<TabItem value="multiple">

```ano
        ...
        mask <column> <rule-name>
            format "%s %s ..."
            file src/main/resources/<file-name-1.txt> random-order // 1st %s
            file src/main/resources/<file-name-2.txt> random-order // 2nd $s
            ...
        ...
```

</TabItem>

</Tabs>

- Specify desired output format string
- Make sure the `.txt` files resides in the folder `src/main/resources/` in the generated project.
- Pick values sequentially from the file (specify nothing) or randomly by adding the keyword `random-order`

### Multiple file inputs

The format strings can take any number of inputs:

- 1st `%s` takes value from "file-name-1.txt", because it is the first specified
- 2nd `%s` takes values from "file-name-2.txt"
- etc.

E.g.: With multiple inputs, the order of input files corresponds to the order of the `%s`Â´s in the `format` string

## File Structure

- One word per line
- Each line is one value
