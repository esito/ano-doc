# `random-date`

:::tip Goal
Write `mask` rule logic to

- generate **random dates** from an interval defined by `<from>` and `<to>` properties
- The `random-date` function generates values having standard format `YYYY-MM-DD`.

:::

## Code Structure

<Tabs
  defaultValue="structure"
  values={[
    { label: 'Structure', value: 'structure' },
    { label: 'Update Example', value: 'update' },

  ]}
>

<TabItem value="structure">

```ano
        ...
        mask <column> <rule-name>
            format %s
            random-date <from> <to>      // [<from>, <to>]
        ...
```

</TabItem>

<TabItem value="update">

```ano
table HOTELROOMCATEGORY
    ...
    column date FROMDATE
    ...
...
task ... {
    update HOTELROOMCATEGORY
        ...
        mask FROMDATE HOTELROOMCATEGORY_FROMDATE
            format %s
            random-date 2020-01-01 2022-12-31
        ...
}
```

</TabItem>

<TabItem value="create">

```ano
table HOTELROOMCATEGORY
    ...
    column date FROMDATE
    ...
...
task ... {
    create HOTELROOMCATEGORY
        minimum-rows 50
        ...
        mask FROMDATE HOTELROOMCATEGORY_FROMDATE
            format %s
            random-date 2020-01-01 2022-12-31
        ...
}
```

</TabItem>

</Tabs>

- `<from>` and `<to>` values are included in the generated values.
  - Both are written using `YYYY-MM-DD format`
- Since `random-date` creates values of the format `YYYY-MM-DD`, the `%s` will result in strings of this format. For different formats, see [Formats](#)
