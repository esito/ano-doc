# How to `format` Results

:::tip Goal
The goal of `format` is to let you write how the anonymization results should be formatted when stored in the database.

It may be a combinaton of literal string values and format strings and uses [java.util.Formatter](https://dzone.com/articles/java-string-format-examples).

**Note:** When a transformation method is defined for a rule, the formatted value is the input to the transformation method. The output of the transformation is then stored in the database.
:::

## Java Formatter Documentation

For a thorough guide on the definiton of different flags (e.g. `%f`, `%tT`, ... ), see [Java String Format Examples](https://dzone.com/articles/java-string-format-examples).

## Deciding the format

The column type in your table decides the format. Checks and specifications of the internal structure of each data value can be customized using the format string, and by writing custom transformation methods.

### Note

- Without any format specifications the value is regarded as a constant.
- The text constant NULL is regarded as SQL NULL.
- Each format specifier must be mached with one input source.

## Integers and Decimals

### Integers

Use `%d`

### Decimals

Either use `%f` or `%.nf`, where `n` is the number of decimal places (e.g. `%.2f` for 2 decimal places).

## Text Strings

Either use `%s` or `"..."`. When using double quotes, you can write any of combination of format specifiers and literals into the string.

Make sure each format specifier is matched with an input source (e.g. [Mask - Input Sources](./mask/overview-mask#types-of-input-sources)).

## Dates, Times and Datetime

### Dates

Dates are usually stored with the ISO 8601 standard format `yyyy-MM-dd`. In this case use the format specifier `%tF`.

If the format is different, check out [Java String Format Examples](https://dzone.com/articles/java-string-format-examples).

### Times

Times are usually stored with the 24-Hour format `HH:mm:ss`. In this case use the format specifier `%tT`.

If the format is different, check out [Java String Format Examples](https://dzone.com/articles/java-string-format-examples).

### Datetimes

Datetimes are usually stored with the database timestamp format `yyyy-MM-dd HH:mm:ss`. In this case use the format specifier `%1\$tF %1\$tT`.

If the format is different, check out [Java String Format Examples](https://dzone.com/articles/java-string-format-examples).
